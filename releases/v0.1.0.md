Githome v0.1.0 is the first complete build: a self-hosted forge you can point
an unmodified `gh` CLI at, clone and push with stock `git`, and get
GitHub-shaped REST v3 and GraphQL v4 responses from. It covers the full path
from an empty database to merged pull requests with signed webhook deliveries.

## What works

**Authentication.** Classic PATs with prefix + CRC32 checksum. OAuth device
flow. Scopes on every response header. Token hash stored, never the token itself.

**Repositories and git read.** `gh repo view`, `git clone`, `git fetch`.
Branches, tags, refs, the commit log, tree/blob contents, compare diffs.
GraphQL `repository(owner, name)` query.

**Git push.** `git push` over Smart HTTP with `receive-pack`. REST ref create
and update. Post-receive sync keeps the database consistent with what git wrote.
`refs/pull/{n}/head` and `refs/pull/{n}/merge` are live refs.

**Issues.** Full lifecycle over REST and GraphQL: create, list, view, comment,
label, milestone, reaction, close, reopen. `gh issue list`, `gh issue view`,
`gh issue create` all work against a live instance.

**Pull requests.** Open from a branch, diff, read mergeable state (computed
async), merge with squash, merge commit, or rebase. `gh pr create`,
`gh pr view`, `gh pr merge`.

**Code review.** Reviews with approve/request-changes/comment, inline comments
anchored to diff hunks, review threads with resolve/unresolve, commit statuses,
check runs, `reviewDecision`, and `statusCheckRollup`.

**Webhooks and the Events API.** Repository webhooks with `X-Hub-Signature-256`
HMAC delivery, SSRF guard, at-least-once retry. `GET /repos/{owner}/{repo}/events`
backed by the same event record that drives deliveries.

**Search.** `/search/issues`, `/search/repositories`, `/search/code` with
qualifier parsing, the `{total_count, incomplete_results, items}` envelope, and
a visibility predicate so private repos never appear in unauthenticated results.

**Pagination and conditional requests.** RFC 5988 `Link` header on every list
endpoint so `gh api --paginate` works. `ETag` on GET responses; a matching
`If-None-Match` returns 304 without spending rate limit.

**GraphQL introspection.** Schema serves the `Query`/`Mutation` roots, the
GitHub-named object types, Relay `PageInfo`, and `first`/`after`/`last`/`before`
on issue and PR connections.

## Install

Download the archive for your platform, verify the checksum, and put the
binary on your PATH:

```sh
# example: linux/amd64
tar -xzf githome_v0.1.0_linux_amd64.tar.gz
sudo mv githome_v0.1.0_linux_amd64/githome /usr/local/bin/
sudo mv githome_v0.1.0_linux_amd64/githome-migrate /usr/local/bin/
```

Or build from source with Go 1.26+:

```sh
go install github.com/tamnd/githome/cmd/githome@v0.1.0
go install github.com/tamnd/githome/cmd/githome-migrate@v0.1.0
```

## Quick start

```sh
# Postgres
export GITHOME_DATABASE_URL="postgres://user:pass@localhost/githome"
export GITHOME_HTML_URL="http://localhost:8080"
githome-migrate up
githome

# SQLite (no external database needed)
export GITHOME_DATABASE_URL="sqlite:githome.db"
githome-migrate up
githome
```

Point `gh` at it:

```sh
export GH_HOST=localhost:8080
gh auth login
gh repo create myorg/myrepo --private
git clone https://localhost:8080/myorg/myrepo.git
```

## Verify a release archive

```sh
sha256sum -c githome_v0.1.0_linux_amd64.tar.gz.sha256
```

## Platform support

Binaries are static (no libc dependency, no cgo) and built from a single
toolchain:

| OS      | Architectures                               |
|---------|---------------------------------------------|
| Linux   | amd64, arm64, arm, 386, ppc64le, s390x, riscv64, loong64 |
| macOS   | amd64, arm64                                |
| Windows | amd64, arm64, 386                           |
| FreeBSD | amd64, arm64                                |
| OpenBSD | amd64, arm64                                |

## Known gaps

This is v0.1.0. A few things are intentionally deferred:

- SSH transport (clone and push over `git@host:owner/repo`)
- Org accounts (everything is user-owned for now)
- At-rest webhook secret encryption
- Full-text search indexes (current implementation uses portable LIKE scans;
  an FTS5/tsvector backend is specced but not wired yet)
- GitHub Actions runner compatibility
