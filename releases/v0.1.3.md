Githome v0.1.3 is the release where the server grows a face. v0.1.0 shipped the
REST and GraphQL forge, v0.1.1 made it fast, and v0.1.2 turned a tag into every
downloadable artifact. This release adds the built-in web UI that those notes
never mentioned, hardens the API against real `gh` traffic with a conformance
suite wired into CI, and fills a long list of compatibility gaps. The API
contract from v0.1.1 is unchanged, so existing clients keep working; everything
here is additive.

## The web UI

The biggest change is a server-rendered web interface, served from the same
binary and on by default. It is a reimplemented Primer look, plain
`html/template` with htmx for the interactive bits and a handful of web
components, and the write paths work with JavaScript disabled. No React, no
build step, no second process.

**Browsing.** Repository home, the file tree, blob view with syntax
highlighting, rendered Markdown, the commit log, a single-commit view, and
blame that annotates each line with the commit that introduced it. Code lines
are selectable and permalink with `#L3-L7` anchors, and short-SHA and numeric
permalinks canonicalize to their full form.

**Pull requests and review.** The conversation, the commits, and a Files tab
that renders the diff unified or side-by-side, highlights word-level changes
inside an edited line, and expands hunk context on demand. The diff display
choice rides a query parameter so a link carries the reader's view.

**Issues and the rest.** Issue threads with milestone resolution and paging for
long conversations, labels and locks and list filters, a dashboard, a checks
tab, a notifications inbox backed by the domain layer, user profiles with stars,
followers, and following tabs, a repository Settings tab for owners, and the
account settings tree. The whole thing scales down to a phone with a real
breakpoint scale and touch targets.

**Auth and browse mode.** Sign in, sign up, and sign out in the browser, `gh
auth login` working end to end through the web flow, and a `githome browse`
subcommand that serves a local repository read-only with no sign-in for
zero-config viewing.

## Compatibility, certified in CI

The definition of done stopped being a claim. `githome-conform` certifies a
running instance against the compatibility matrix as a black-box client and
exits non-zero on any drift, and that conformance run is now wired into CI
alongside contract tests that replay the literal GraphQL documents `gh` sends.

**GraphQL.** The surface now validates the documents the `gh` CLI actually
emits. Query cost is priced by GitHub's node-count model rather than a fixed
point cap, so a large-but-bounded query passes where it used to be rejected.
Rate-limit headers, `X-RateLimit` accounting that refunds on a 304, honest
errors instead of silently dropped or faked mutations, the write-side mutations
`gh` sends, commit data fields, `mergeCommit` and `closingIssuesReferences`,
the `license`, `codeOfConduct`, and meta root queries, and a long list of
filled Issue, Label, User, and Repository fields.

**REST.** The P0 and P1 compatibility ladders: the merge check, the root
document, contents `sha`, branch protection, notifications, pagination, checks,
refs, releases, gists, org membership and webhooks, search, the event-feed
polling contract, branch merge, `repository_dispatch`, lookup-by-id, the
`text-match` media type on search, a real collaborator surface that honors
roles for repository write access, global repository listing, the community
profile, CODEOWNERS errors, and a webhook test endpoint.

**Releases, gists, and apps.** A full Releases API with asset upload at
`/api/uploads/`, full Gist CRUD with stars, forks, and comments, and the OAuth
web flow, device flow, and GitHub Apps endpoints.

## Performance and serving

The second rung of the performance ladder takes the read path off its worst
hotspots, brings page renders inside the budget, trims per-render allocations
on the hot template path, pools the render buffer, and streams assets instead
of buffering them. The server also speaks HTTP/2 cleartext (h2c).

## Storage correctness

The Postgres leg of the test matrix is now green and stays that way. A drift
guard asserts the consolidated `schema.pg.sql` matches a full migration replay,
the rollback of the GitHub Apps migration drops its foreign-key columns in the
right order, and code search matches paths and orders results the same way on
Postgres as on SQLite. The suite runs against both SQLite and Postgres on every
push.

## Upgrading from v0.1.2

The HTTP and GraphQL contract is unchanged. Migrations run automatically on
startup (or `githome-migrate up` if you run them yourself), so the upgrade is a
drop-in. The web UI is on by default; set `GITHOME_WEB_ENABLED=false` to serve
the APIs and git transport only.

## Install

See [INSTALL.md](https://github.com/tamnd/githome/blob/main/INSTALL.md) for every
method. The common paths:

```sh
brew install tamnd/tap/githome                          # macOS
docker pull ghcr.io/tamnd/githome:0.1.3                  # container
go install github.com/tamnd/githome/cmd/githome@v0.1.3   # from source
```

Or download an archive and verify it:

```sh
tar -xzf githome_0.1.3_linux_amd64.tar.gz
sha256sum -c checksums.txt
sudo install githome/githome /usr/local/bin/
```

## Platform support

Binaries are static (no libc dependency, no cgo) and built from a single
toolchain:

| OS      | Architectures                              |
|---------|--------------------------------------------|
| Linux   | amd64, arm64, arm, 386, ppc64le, s390x, riscv64 |
| macOS   | amd64, arm64                               |
| Windows | amd64, arm64, 386                          |
| FreeBSD | amd64, arm64                               |
| OpenBSD | amd64, arm64                               |

## Known gaps

Unchanged from v0.1.2:

- SSH git transport (the config is reserved, the server is not wired)
- Organization accounts (everything is user-owned for now)
- At-rest encryption of webhook secrets
- GitHub Actions runner compatibility (in progress in the separate
  tamnd/githome-action project, not yet served by the githome binary)
