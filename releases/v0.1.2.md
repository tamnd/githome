Githome v0.1.2 is a packaging release. The server is the same as v0.1.1: an
unmodified `gh` CLI, stock `git`, GitHub-shaped REST v3 and GraphQL v4. What
changed is how you get it. v0.1.0 and v0.1.1 shipped as hand-built archives;
from here on one tag push produces every artifact a package manager needs, all
from a single GoReleaser config.

## What changed

**One release pipeline.** A new `.goreleaser.yaml` turns a `vX.Y.Z` tag into the
full set of artifacts in one run: per-platform archives, Linux packages, a
multi-arch container image, checksums, an SBOM, and a cosign signature. The
build stays pure Go with cgo disabled, the same static cross-compile the build
matrix already proves, so every binary is libc-free and reproducible.

**Linux packages.** Each release now carries `.deb`, `.rpm`, and `.apk` packages
built with nfpm (no dpkg, rpmbuild, or abuild needed on the runner). The deb and
rpm drop a hardened `githome.service` systemd unit and a commented
`/etc/githome/githome.env`, and the postinstall script provisions the service
account and data directory, so `apt install ./githome_*.deb` leaves a unit ready
to enable. Alpine has no systemd, so the apk ships the binaries and the env file
only.

**Container image.** A multi-arch image (`linux/amd64`, `linux/arm64`) is pushed
to `ghcr.io/tamnd/githome` on every release, tagged with the version and
`latest`. It is built on Alpine rather than distroless because the server shells
out to a real `git` binary for the smart-HTTP transport, and the manifests are
cosign-signed.

**Package managers.** The pipeline also generates entries for Homebrew (a macOS
cask), Scoop, winget, the AUR, and Nix. Each one pushes to a repository that
lives outside this project, so each self-disables until its token is configured.
A release with no extra secrets still produces every downloadable artifact and
the container image, and each manager lights up the moment its repository and
secret exist. INSTALL.md lists exactly which repository and which secret each
one wants.

**Supply-chain metadata.** Every release attaches a `checksums.txt`, a syft SBOM
per archive, and a keyless cosign signature over the checksums, signed with the
workflow's OIDC token. The container manifests are signed the same way.

## Install

See [INSTALL.md](https://github.com/tamnd/githome/blob/main/INSTALL.md) for every
method. The common paths:

```sh
brew install tamnd/tap/githome                          # macOS
docker pull ghcr.io/tamnd/githome:0.1.2                  # container
go install github.com/tamnd/githome/cmd/githome@v0.1.2   # from source
```

Or download an archive and verify it:

```sh
tar -xzf githome_0.1.2_linux_amd64.tar.gz
sha256sum -c checksums.txt
sudo install githome*/githome /usr/local/bin/
```

## Platform support

Binaries are static (no libc dependency, no cgo) and built from a single
toolchain:

| OS      | Architectures                              |
|---------|--------------------------------------------|
| Linux   | amd64, arm64, arm, 386, ppc64le, s390x, riscv64 |
| macOS   | amd64, arm64                               |
| Windows | amd64, arm64, 386                          |
| FreeBSD | amd64, arm64                               |
| OpenBSD | amd64, arm64                               |

## Known gaps

Unchanged from v0.1.1:

- SSH transport (clone and push over `git@host:owner/repo`)
- Org accounts (everything is user-owned for now)
- At-rest webhook secret encryption
- GitHub Actions runner compatibility
- An apt/yum repository for `apt install githome` straight from a hosted repo
  (the `.deb` and `.rpm` are attached to each release for now)
